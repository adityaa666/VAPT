DOCKER CONTAINER SECURITY
WHAT IS DOCKER?
Docker is containerization platform that packages your application and dependencies together in form of docker container to ensure that your application works seamlessly in any environment
Docker container is standardized unit which can be created on fly to deploy a particular application or environment
It could be Ubuntu container, CentOS container etc.
Benefits of docker
Speed
Portability
Efficieny
CONTAINER vs VM
Containers are app level construct
VMs are an infrastructure level construct to turn one machine into many servers
DOCKER SECURITY FEATURES
CGROUPS
Linux kernel feature that provides capability to restrict resources like cpu, memory, io, network bandwidth among set of processes
Docker allows to create containers using cgroup which allows for resource control for specific container
Command: docker run -it -m 500M --kernel-memory 50M --cpu-shares 512 --blkio-weight 400 --name ubuntu
CAPABILITIES
Root user has all privileges enabled
Capabilities allow finer control for capabilities that can be allowed for root user
In Linux 3.19.0.21, there are 36 capabilities
File: "/usr/include/linux/capability.h"
Docker turns on only 14 capabilities by default for containers started with default options
Capabilities like insert/remove kernel modules, system clock manipulation are blocked

CROSS SITE SCRIPTING (XSS)
WHAT IS XSS?
Flaw occur when application includes untrusted data in new webpage without validation or escaping or updates existing page with user-supplied data using browser app that create HTML or Javascript
CSS is client side code injection attack
Attacker uses vulnerable page to deliver malicious Javascript to user side
Client-side code injection attack where malicious scripts are injected into trusted websites
Occurs in over in those web-applications where input-parameters are not properly sanitized or validated which thus allows an attacker to send malicious Javascript codes over at different end-user
WHAT IS JAVASCRIPT?
Module for dynamic web application developments
Javascript is programming language for web
Javascript handle update and change both HTML and CSS
Calculate, manipulate and validate data
JAVASCRIPT EVENT HANDLERS
Handle and verify user input, user actions and browser actions
Example, onload, onmouseover, onclick, onload, onmouseover, onmouseout, onunload
IMPACT OF CROSS-SITE SCRIPTING
Capture and access authenticated session
Uploads phishing page to lure users into unintentional actions
Redirects the visitors to some other malicious sections
Expose the user's sensitive data
Manipulate the structure of web application or even defaces it
TYPES OF XSS
Reflected XSS
Arises when application receives data in HTTP request and includes that data within immediate response in unsafe way
Stored XSS
Arises when application receives data from untrusted source and includes data within its later HTTP response in unsafe way
DOM Based XSS
Arises when JS takes data from attacker-controllable source such as URL and passes it to sink that supports dynamic code execution such as eval() or innerHTML
Enables attacker to execute malicious JS, which typically allows them to hijack other users account
XSS MITIGATION STEPS
Developer to implement whitelist of allowable inputs and if not possible then there should be some input validations and data entered by user must be filtered as possible
Output encoding is most reliable solution to combat XSS .. takes up script code and converts it into plain text
WAF should be implemented as it somewhere protects the application from XSS attacks
Use of HTTPOnly flags on cookies
Developers can use Content Security Policy (CSP) to reduce XSS
CROSS SITE REQUEST FORGERY (CSRF)
CSRF BASICS
Cookies
Arbitrary pieces of data chosen first and sent by web server and stored on client computer by web browser
Session ID
Piece of data used in network communications to identify a session, a series of related message exchages
Necessary in cases where communications infrastructure uses a stateless protocol such as HTTP
Same Origin Policy
Web browser permits scripts contained in first web page to access data in second web page having same origin
WHAT IS CSRF?
Attack that forces an end user to execute unwanted actions on application in which they are already authenticated
Allows attacker to partly circumvent the same origin policy, designed to prevent different websites from interfering with each other
Attacker causes victim user to carry out action unintentionally
HOW CSRF WORKS?
Related action
Cookie based session handling
No unpredictable request parameters
IMPACT OF CSRF
Change victim email address or password
Change cart items or make bank transcations
Explicitly logging out the user from account
MITIGATION FOR CSRF
Anti-CSRF token
Same-Site cookies attributes
Disable GET request for state-change
Content Security Policy (CSP) set
Source origin and One-Time token
WEB APPLICATION FIREWALL (WAF)
WAF Architecture
WHAT IS WAF?
Form of application firewall that filters, monitors and blocks HTTP traffic to and from web service
IMPORTANT TERMS
Reverse Proxy
Reverse proxy server is intermediate connection point positioned at network edge
Receives initial HTTP connection request acting like actual endpoint
Forward Proxy
Forward proxy is mostly called "proxy"
Send connection request to it and forward proxy retrieves data from internet
Usually lets clients on otherwise firewall restricted network to access the internet
Policies
WAF operates through set of rules
Blacklist
Involves defining which entities should be blocked
Whitelist
Practice of explicitly allowing some identified entities access to privilege actions, service, mobility or recognition
Transparent Reverse Proxy
Reverse proxy with transparent mode
WAF sends filtered traffic to web applications
Transparent Bridge
HTTP traffic goes directly to web application
WAF FEATURES
Load Balancing and Global server load balance
Avoid browsing limits
Avoid block content access
Protect identity
TYPES OF WAF
Network based WAF
Hardware that installs on device in order to track all data packets to and from website
Host based WAF
Host based WAF exists as modules for web server
Cloud based WAF
Provide similar benefits as other software-based WAF solutions such as low cost and lack of on-premises resources that you manage
HOW WAF WORKS?
Using set of rules to distinguish between normal requests and malicious requests
Sometimes they use learning mode to add rules automatically through learning about user behaviour
WAF OPERATING MODES
Negative Model (Blacklist based)
Blacklisting model uses pre-set signatures to block web traffic that is clearly malicious
Signatures designed to prevent attacks which exploit certain websites and web application vulnerabilities
Positive Model (Whitelist based)
Whitelisting model only allows web traffic according to specifically configured criteria
Configured to only allow HTTP,GET requests from certain IP address
Mixed/Hybrid Model (Greylist or Inclusive Model)
Hybrid security blends both whitelisting and blacklisting

SQL INJECTION
WHAT IS SQL INJECTION?
Code injection technique used to attack data driven application in which malicious SQL statements are inserted into entry field for execution
Injection attacks trick application into including unintended commands in data send to an interpreter
Successful SQL injection exploit can read sensitive data from database, modify database data, execute administration operations on database, recover the content of given file present on DBMS file system and in some cases issue commands to OS
HOW SQL INJECTION WORKS?
Form sent to user
Attacker submits form with SQL exploit data
Application builds string with exploit data
Application sends SQL query to DB
DB executes query, including exploit, sends back to application
Application returns data user
